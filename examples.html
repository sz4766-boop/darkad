Case 1: Speed abuse via client authority

The attacker modifies local movement values (or prediction code) to exceed limits. If the server accepts client-reported position or velocity without sanity checks, the game becomes unfair and hard to moderate.

Mitigation pattern: server-authoritative movement, reconcile prediction, cap deltas, and flag impossible accelerations.

Case 2: Packet replay / order manipulation

The attacker records valid requests and replays them later (or reorders them) to confuse transitions and bypass cooldowns. This is common in systems that trust “the last packet wins” without a strict state machine.

Mitigation pattern: nonces, server-issued sequence numbers, short-lived tokens, and state-machine validation.

Case 3: Aim assistance via external tooling

External tools can read process memory or interpret rendering information to provide unnatural accuracy. Even without packet modification, behavior becomes statistically implausible.

Mitigation pattern: behavior-based detection, server-side visibility checks, and layered anti-tamper (balanced with false positives).
